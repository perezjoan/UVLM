UVLM — Version History
======================

v3.0.0 (April 2026) — Package release
--------------------------------------
- Refactored: Monolithic Colab notebook into pip-installable Python package
  (uvlm/) with 8 modules: loader, inference, parsers, consensus, batch,
  prompts, registry, utils.
- Added: Local Jupyter notebook interface (notebooks/UVLM_local.ipynb) for
  running UVLM on local GPU hardware without Google Colab.
- Added: Google Colab notebook interface (notebooks/UVLM_colab.ipynb) that
  installs the uvlm package from GitHub automatically.
- Added: Programmatic API — load_model(), run_inference(), parse_response(),
  run_batch() can be called from any Python script without notebooks.
- Changed: All global state eliminated. Model, processor, and backend are
  passed as a model_ctx dict returned by load_model().
- Changed: run_inference() now returns (raw_response, token_count) tuple
  instead of storing token count in a global variable.
- Added: Environment detection (is_colab()) for automatic path and token
  handling across Colab and local deployments.
- Added: Unified HF token retrieval via get_hf_token() — tries explicit
  argument, Colab secrets, HF_TOKEN env var, and huggingface-cli cache.
- Added: pyproject.toml for pip installation from GitHub.
- Added: .gitignore for Python/Jupyter artifacts.
- Preserved: All inference, parsing, consensus, and batch logic unchanged
  from v2.2.2. No behavioral changes.

v2.2.2 (March 2026) — Initial public release
---------------------------------------------
- Added: Truncation detection for all task types using exact generated token
  count from model output.
  - New CSV column: {col}_truncated.
  - Console warning displayed when truncation is detected.
- Updated: Advanced reasoning mode automatically overrides token limit to
  ADVANCED_REASONING_MAX_TOKENS = 1024.
- Updated: Max tokens slider range extended to 1500 to allow user-defined
  reasoning budgets.
- Updated: Consensus validation runs extended to configurable range of 2–5.
- Fixed: Reasoning output column ({col}_reasoning) is no longer truncated.
- Fixed: UTF-8 encoding issues when writing CSV outputs.
- Added: Pre-configured benchmark notebooks with dynamic output filenames.

v2.2.1 (February 2026)
-----------------------
- Fixed: NA value handling in consensus validation. The compute_consensus()
  function now properly filters out NA values (failed parses) before majority
  voting, ensuring that parsing failures do not influence the final answer.
  Agreement ratio is still computed over all runs (including NA) to preserve
  the reliability metric.
- Added: is_na_value() helper recognizing "NA", "N/A", "NAN", "NONE", "NULL",
  empty strings, and None values.

v2.2 (January 2026)
--------------------
- Added: Advanced reasoning mode (chain-of-thought prompting).
  - Per-task checkbox to enable structured CoT prompting.
  - Auto-generated format instruction with "ANSWER: <value>" pattern.
  - Max token budget override: 1024 tokens (vs default 50).
  - Response parser scans last 5 lines for ANSWER: pattern.
  - Graceful fallback to standard parsing if format not followed.
  - Reasoning trace stored in {column}_reasoning CSV column.
- Added: Disabled advanced reasoning for "text" task type.

v2.1 (December 2025)
---------------------
- Added: Consensus validation feature.
  - Per-task toggle with configurable runs (2 or 3, default: 2).
  - Majority voting across all runs (equal peer voting).
  - Percentage-based numeric tolerance for grouping similar values.
  - Output columns: {col}_consensus, {col}_agreement, {col}_runs.
  - Consensus not available for "text" task type.

v2.0 (November 2025)
---------------------
- Added: Multi-task prompt builder with widget-based UI (up to 10 tasks).
- Added: Four task types: numeric, category, boolean, text.
- Added: Four-element prompt structure: role, task, theory, format.
- Added: Batch execution engine (Block 3) with image folder iteration.
- Added: Resume mode: skip already-processed images.
- Added: CSV schema upgrading: append new task columns to existing CSV.
- Added: Per-task error handling (errors logged as "NA").
- Added: Periodic checkpoint saves (every 5 images).

Note
----
Versions prior to **v2.2.2** were developed in a **private repository**.
The public GitHub release history therefore begins at **v2.2.2**.
